# 快速开始

这一页只做一件事：

让你在 5 分钟内用 `Netunnel` 跑通第一次 HTTP 隧道，并拿到一个可访问的公网地址。

如果你还没有安装客户端，先进入 [下载](../download/README.md) 页面完成安装。

## 适合谁

这份快速开始最适合这些场景：

- 想把本地 Web 页面临时发布到公网
- 想把本地 API 给前端、测试或同事联调
- 想先体验一次 `Netunnel` 是否适合自己

## 开始前准备

在开始前，先准备这 3 样东西：

1. 一台已经安装好 `Netunnel` 客户端的 Windows 电脑
2. 一个你本地已经跑起来的 Web 服务或 API
3. 一个可以正常登录的账号

如果你只是为了测试，可以先在本地启动一个简单页面或本地接口服务。

## 目标结果

完成这份快速开始后，你应该拿到：

- 一个已经创建成功的 `HTTP` 隧道
- 一个可以从外网访问的公网地址
- 一次完整的首次体验

## 第一步：下载并登录客户端

1. 从 [下载](../download/README.md) 页面下载 Windows 客户端
2. 安装并打开客户端
3. 登录你的账号

完成后，你应该能进入客户端主界面。

## 第二步：准备本地服务

在创建隧道前，先确认你本地确实有一个可访问的服务。

例如：

- 本地前端页面运行在 `http://127.0.0.1:3000`
- 本地 API 运行在 `http://127.0.0.1:8080`

建议先在本机浏览器里确认这个地址能打开，再继续下一步。

## 第三步：创建 HTTP 隧道

1. 在客户端里进入隧道创建入口
2. 选择 `HTTP` 隧道
3. 填写本地服务地址或本地端口
4. 提交创建

这一版文档先不写过细字段说明，目标是先帮助用户跑通流程。后续可以在 [使用文档](../docs/README.md) 里拆成更详细的隧道配置说明。

## 第四步：复制公网访问地址

创建成功后，客户端应提供一个公网访问地址。

你现在要做两件事：

1. 复制这个地址
2. 在另一台设备、浏览器无痕窗口或手机上尝试访问

如果可以访问成功，说明第一次体验已经跑通。

## 第五步：验证是否成功

成功的标准不是“隧道创建成功”，而是“公网地址真的能访问”。

建议这样验证：

1. 在本机先确认本地服务本身可访问
2. 再访问 `Netunnel` 给出的公网地址
3. 最好换一个外部环境再访问一次，例如手机流量或另一台电脑

## 如果你只想最快跑通一次

第一次体验建议优先这样选：

- 优先选 `HTTP` 隧道
- 优先映射一个本地网页或简单 API
- 不要一上来就用复杂 TCP 场景
- 先确认本地服务正常，再创建隧道

这样最容易在最短时间内得到正反馈。

## 常见卡点

首次体验通常会卡在下面几步：

- 客户端安装后打不开
- 登录失败
- 本地服务本身没有启动
- 隧道创建成功，但公网地址访问失败

这些问题后续会逐步补充到 [常见问题](../faq/README.md) 页面。
