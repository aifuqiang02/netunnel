# 常见问题

这一页优先解决第一次使用 `Netunnel` 最容易遇到的问题。

如果你还没有完成第一次体验，建议先看 [快速开始](../quickstart/README.md)。

## 1. 客户端安装后打不开怎么办

先按这个顺序排查：

1. 确认你下载的是 Windows 安装包，而不是其他平台文件
2. 优先使用官网推荐的 `x64-setup.exe`
3. 安装完成后重新双击启动一次
4. 如果系统弹出安全提示，先确认是否被系统拦截
5. 如果依然打不开，重新下载安装包再试一次

如果后续需要，可以在这里继续补充更具体的系统兼容性说明和截图。

## 2. 登录失败怎么办

先检查这几件事：

1. 当前网络是否正常
2. 输入的账号信息是否正确
3. 客户端当前连接的服务地址是否正确
4. 是否存在临时服务异常

如果你能打开客户端但始终无法登录，建议后续补充：

- 登录报错文案示例
- 服务端异常时的用户提示
- 网络不可用时的提示方式

## 3. 设备不在线怎么办

设备不在线，通常说明客户端或设备侧没有完成可用连接。

建议这样排查：

1. 确认你已经在客户端中添加或选择了设备
2. 确认当前设备状态不是离线或异常
3. 如果设备依赖本地 agent，确认 agent 已正常启动
4. 重新打开客户端后再检查一次设备状态

如果后续产品里有更明确的设备状态说明，这里可以补成更细的排查表。

## 4. 本地服务正常，但公网地址打不开怎么办

这是首次体验最常见的问题之一。

建议按下面顺序检查：

1. 先在本机访问本地服务地址，确认本地服务本身是好的
2. 再检查创建隧道时填写的本地地址和端口是否正确
3. 确认你创建的是合适的隧道类型，例如网页或 API 优先用 `HTTP`
4. 复制客户端给出的公网地址，重新在外部环境访问一次
5. 如果本机能访问，本地服务也正常，但公网地址不通，再回头检查隧道配置和设备状态

一个很常见的误区是：

- 以为“隧道创建成功”就等于“公网一定能访问”

真正的成功标准是：

- 你在外部环境里真的打开了那个公网地址

## 5. 第一次应该选 HTTP 还是 TCP

第一次体验，优先选 `HTTP`。

推荐这样理解：

- 如果你要发布网页、本地前端页面、本地 API，优先选 `HTTP`
- 如果你要映射远程桌面、数据库、其他 TCP 服务，再考虑 `TCP`

为什么第一次建议先选 `HTTP`：

- 更容易理解
- 更容易验证是否成功
- 更适合作为首次体验路径

## 6. 下载页里有多个安装包，我该下哪个

如果你是普通 Windows 用户，优先下载：

- `Netunnel.Desktop_2.12.11_x64-setup.exe`

如果你需要 MSI 安装方式，可以使用：

- `Netunnel.Desktop_2.12.11_x64_en-US.msi`

如果你不确定，默认选 `x64-setup.exe` 就够了。

## 7. 第一次体验应该选什么场景

最适合第一次体验的场景是：

- 把本地网页发布到公网
- 把本地 API 暴露给前端或测试联调

不建议第一次就上来做：

- 复杂 TCP 服务
- 多层网络环境的远程访问
- 不确定本地服务是否正常的场景
