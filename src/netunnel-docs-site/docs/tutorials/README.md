# 场景教程

这里是 `Netunnel` 的场景教程入口。

这一部分主要面向两类人：

- 还没决定是否使用 `Netunnel`，但正在寻找解决方案的人
- 已经开始体验 `Netunnel`，但需要针对具体场景找到操作路径的人

和“使用文档”不同，这一部分不按功能模块组织，而按用户真实问题组织。

## 这部分内容承担什么作用

场景教程会同时承担这些任务：

- 承接搜索流量
- 承接知乎、B 站、掘金、社群等外部推广内容
- 帮助用户快速判断 `Netunnel` 能不能解决自己的问题
- 通过具体任务降低首次使用门槛

## 第一版优先做的 4 类场景

### 1. 本地接口联调

适合的问题：

- 本地 API 怎么给前端联调
- 没有公网环境，怎么让测试访问本地接口
- 如何把本地服务短时间暴露给其他人

建议后续优先写的文章方向：

- [把本地 API 发布到公网进行联调](./local-api-public-debug.md)
- 前端、测试、第三方如何访问你的本地接口
- 用 `HTTP` 隧道做本地开发联调

### 2. 回调调试

适合的问题：

- 微信支付回调本地怎么调试
- Webhook 本地怎么接收
- 飞书、企业微信、第三方平台回调如何调试

建议后续优先写的文章方向：

- [微信 / 支付 / Webhook 回调本地调试](./webhook-callback-local-debug.md)
- 使用 `Netunnel` 接收本地 Webhook
- 没有公网服务器时如何调试第三方回调

### 3. 临时页面演示

适合的问题：

- 怎么把本地页面临时给客户看
- 演示版页面还没上线，怎么快速给别人访问
- 同事需要看你本地页面，怎么最快给链接

建议后续优先写的文章方向：

- 本地页面临时发布到公网
- 给客户演示本地项目的最短路径
- 不上正式服务器，怎么让别人访问你的本地网页

### 4. 远程访问设备

适合的问题：

- 没有公网 IP，怎么访问家里电脑
- 怎么远程访问 NAS
- 如何暴露本地 TCP 服务到公网

建议后续优先写的文章方向：

- 远程访问家里电脑的轻量方案
- 使用 `TCP` 隧道访问远程服务
- 如何把家里的轻量服务映射到公网

## 推荐内容写法

为了更适合推广和转化，建议这部分内容统一按下面结构写：

1. 用户遇到的问题是什么
2. 为什么会卡在这个问题上
3. 用 `Netunnel` 的最短解决路径是什么
4. 具体操作步骤
5. 常见错误和排查方式
6. 适合什么人继续使用

也就是说，先解决问题，再介绍产品，不要反过来写。

如果现在只先写 2 篇教程，我建议优先写：

1. `把本地 API 发布到公网进行联调`
